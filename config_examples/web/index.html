It works!



This is the default web page for the VSCP daemon. This is a fully working web server so you can add to HTML files to the folder you have setup using <webrootpath> in the VSCP configuration file and serve the as from any other webserver. /vscp is a reserved path and you should not us it in your own folder structure as it will take you to this reserved page.

To test the websocket interface of the VSCP daemon you first edit the setup.js file which after installation is located in the testws subfolder of the web root filesystem.

The default content in this file is:

var vscp_setting_user = "admin";
var vscp_setting_authdomain = "mydomain.com";
var vscp_setting_passwordhash = "d50c3180375c27927c22e42a379c3f67";

which is the default admin credentials for

Username: admin
Password: secret

needless to say you should change this as soon as you use your system more seriously.

The other part in the setup.js file to change is the url for your server. When you have edited and saved this information you can test the websocket login test script. If you see something like

Authentication 1 - Logged in

at the end of the output from this scripts everything is OK, if not something is bad with your login information.

p> The default for the websocket samples is non-ssl. If you want to use ssl (with the ssl version of the VSCP daemon) you should use "wss:/" as prefix instead of "ws://
